Qué tomamos de Moldable Development

Moldable object

Repo, módulo, fichero, clase, función, route, PR o trace se vuelven objetos inspeccionables.

Contextual view

El objeto muestra vistas específicas: dependencias, impacto, tests, runtime, connascence.

Contextual action

Acciones útiles: exportar Mermaid, abrir en IDE, generar reporte, registrar ADR.

Contextual search

La búsqueda cambia según el objeto: callers en función, módulos afectados en PR, rutas en repo.

Flujo principal

System Overview
-> Select Scope
-> Apply Lens
-> Inspect Evidence
-> Ask Agent
-> Export / Decide

Esto traduce el mantra de visualización: overview primero, zoom/filtro después, detalles bajo demanda y extracción final.
