Propuesta de producto

MCP como capacidades, agente como guía, UI como inspector

La aplicación más útil no sería un chat ni un grafo gigante. Sería un inspector moldable donde el developer selecciona un scope, aplica un lente, ve evidencia y termina con una decisión.

Arquitectura mental

RepoCódigo, docs, tests, git, runtime.
GraphEntidades, relaciones, comunidades, cambios.
MCPTools, resources, prompts, roots, sampling.
AgentInterpreta, pregunta, explica, propone.
ViewsC4, Mermaid, tablas, grafos, heatmaps.
DecisionRefactor, test, ADR, review, investigar.

Tres propuestas

A. MCP-first

Servidor MCP con herramientas de indexado, consulta, impacto, diagramas y decisiones. Rápido de integrar con IDEs, pero UX chat-first.

B. Visual app

Aplicación dedicada que actúa como cliente/host MCP. Mejor experiencia moldable, más coste de producto.

C. Híbrido IDE + workbench

IDE para flujo diario, workbench visual para inspección profunda. Mejor balance para empezar y escalar.
