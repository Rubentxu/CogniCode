Query strategy

No inventes Cypher: inventá la capa que el usuario necesita

La app necesita un lenguaje de exploración orientado a scopes, lentes, vistas y decisiones. Por debajo puede usar Cypher/GQL/Datalog/DataFusion. Por arriba tiene que hablar como piensa el developer.

show hotspots

Este lenguaje no intenta reemplazar al grafo. Intenta convertir intención de producto en consultas, lentes y vistas.

Decisión técnica

Para MVP: usar un backend property graph con Cypher, probablemente Kuzu si aceptar C++ embebido es razonable. Encapsularlo detrás de un trait Rust GraphStore para no casarse con un motor.

Si el requisito de Rust puro es no negociable, prototipar Grafeo/OCG/SparrowDB/Ladybug, pero sin bloquear el diseño del lenguaje.

Fuentes

Kuzu: graph database embebida con Cypher, ejecución vectorizada/factorizada y bindings Rust.

SurrealDB graph traversal: inspiración de sintaxis por flechas.

Apache DataFusion: motor Rust extensible para analítica y planes custom.

Souffle Datalog: base para reglas de análisis estático.

GT Contextual Search: búsquedas contextuales sobre objetos de dominio.
