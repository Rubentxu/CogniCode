# Known Symbol Inspection Before Spotter

The first technical cut of the CogniCode Explorer MVP will be known-symbol inspection, not Spotter search. This validates the core moldable workflow first: open a known `Symbol` inspectable object, switch contextual views, follow typed call relations, expose evidence blocks, preserve the exploration path, and generate a decision artifact. Spotter remains the natural entry point, but it becomes Phase 1B because search depends on index reliability, ranking, disambiguation, and graph freshness.
