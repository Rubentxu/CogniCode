### CGR_SEC_CRY_001: hardcoded-credentials

| Field | Value |
|-------|-------|
| **rule_id** | `CGR_SEC_CRY_001` |
| **concept_id** | `CC_SEC_CRY_001` |
| **domain** | Security |
| **category** | Credentials |
| **languages** | rust, python, javascript, typescript, go, java, c, cpp |
| **severity** | BLOCKER |
| **precision target** | high |
| **sonarqube_equivalent** | S2068 |
| **cwe_ids** | CWE-798, CWE-312 |

**Detection Strategy**: AST query + token regex hybrid
- Tree-sitter query for string literals assigned to credential-named variables
- Regex for `password`, `secret`, `token`, `api_key`, `apikey`, `credential`, `auth`, `passwd`, `pwd`

**required_keywords**:
```rust
layer! { "preflight" }
required_keywords! { "password", "secret", "token", "api_key", "credential" }
```

**AST Query Pattern** (Rust example):
```tree-sitter
(assignment
  left: (identifier) @var_name
  right: (string_literal) @value)
```

**Preflight Checks**:
1. Skip commented lines (//, ///, /*, #, <!--)
2. Skip test files (*_test.rs, test_*.py)
3. Skip variable names in allowlist: `DEFAULT_PASSWORD`, `EXAMPLE_PASSWORD`, `TEST_PASSWORD`
4. Skip string values < 4 characters (too short to be real credentials)

**False-Positive Conditions**:
- Variable names that are reserved for examples/docs: `DEFAULT_PASSWORD`, `PLACEHOLDER`
- Test fixtures with `#[cfg(test)]` module context
- String concatenation with environment variable: `std::env::var("PASSWORD")`

**Exclusions**:
- Strings in comments/documentation
- Test code under `#[cfg(test)]`
- Values matching `^(?i)(default|example|placeholder|test| dummy)`
- Environment variable access patterns

**Axiom Metadata**:
```yaml
event_kind: security.credentials.hardcoded
severity_mapping:
  BLOCKER: [high, critical]
fix_playbook: |
  1. Identify the hardcoded credential
  2. Replace with environment variable: std::env::var("CRED_NAME")
  3. Or use secrets management: AWS Secrets Manager, HashiCorp Vault
  4. Update documentation to reflect new retrieval method
  5. Verify deployment scripts set the secret correctly
```

**agent_semantics**:
```yaml
summary: "Hardcoded credentials detected - credentials should never be embedded in source code"
reasoning_model: "gpt-4o"
agent_actions:
  - "Search for similar credential patterns in the codebase"
  - "Check for other hardcoded secrets using grep for 'password', 'secret', 'token'"
  - "Verify if credentials are also in version control history"
safe_autofix: false
autofix_guidance: "Cannot safely autofix - requires understanding of credential purpose and replacement strategy"
review_questions:
  - "Is this credential used in production or only development?"
  - "Has this credential been committed to version control?"
  - "What is the appropriate replacement (env var, secrets manager)?"
RAG_chunks:
  - "Hardcoded credentials are a critical security vulnerability (CWE-798)"
  - "The credential was found at ${location} with value pattern ${pattern}"
  - "Fix: Replace with ${correct_approach}"
```

**Performance Budget**: 2ms/file (AST query + regex)
**Complexity**: low

### CGR_SEC_CRY_002: weak-crypto-hash

| Field | Value |
|-------|-------|
| **rule_id** | `CGR_SEC_CRY_002` |
| **concept_id** | `CC_SEC_CRY_002` |
| **domain** | Security |
| **category** | Cryptography |
| **languages** | rust, python, javascript, typescript, go, java, c, cpp |
| **severity** | CRITICAL |
| **precision target** | high |
| **sonarqube_equivalent** | S4792 |
| **cwe_ids** | CWE-327 |

**Detection Strategy**: Tree-sitter AST query
- Match import/use of MD5, SHA1 hash functions
- Check for explicit algorithm identification in hash contexts

**required_keywords**:
```rust
layer! { "preflight" }
required_keywords! { "md5", "sha1", "MD5", "SHA1" }
```

**AST Query Patterns**:

*Rust (using `md5` crate)*:
```tree-sitter
(use_declaration
  path: (scoped_identifier
    name: (identifier) @name
    (#match? @name "^Md5$")))
```

*Python*:
```tree-sitter
(import_statement
  (dotted_name) @name
  (#match? @name "^hashlib$"))
(assignment
  value: (call
    function: (attribute
      object: (identifier) @obj
      attr: (identifier) @attr)
    (#eq? @obj "hashlib")
    (#any-of? @attr "md5" "sha1")))
```

*JavaScript/TypeScript*:
```tree-sitter
(import_statement
  (import_clause
    (named_imports
      (import_specifier
        (identifier) @name)))
  (#any-of? @name "createMD5" "md5" "createSha1" "sha1"))
(call_expression
  function: (member_expression
    object: (identifier) @obj
    property: (identifier) @prop)
  (#eq? @obj "require")
  (#eq? @prop "md5"))
```

**Preflight Checks**:
1. Skip non-security contexts (file checksum, non-cryptographic use)
2. Check if hash is used for checksums (acceptable) vs security (not acceptable)
3. Allow SHA1 for git commit identification (not a security boundary)

**False-Positive Conditions**:
- MD5/SHA1 used for file checksum/checksums (non-security purpose)
- Documentation/examples showing deprecated algorithms
- Comments describing why algorithm is used

**Exclusions**:
- Use in `git` context (commit IDs)
- File integrity checksums (mention "checksum" in nearby code)
- Test fixtures

**Axiom Metadata**:
```yaml
event_kind: security.cryptography.weak_hash
severity_mapping:
  CRITICAL: [high, critical]
fix_playbook: |
  1. Identify the use case (password hashing, integrity check, etc.)
  2. For password hashing: use bcrypt, Argon2, or scrypt
  3. For integrity: SHA-256 or SHA-3 minimum
  4. Update dependencies if using md5/sha1 crates
  5. Run tests to verify functionality unchanged
```

**agent_semantics**:
```yaml
summary: "Weak cryptographic hash (MD5/SHA1) detected - these provide insufficient security for modern applications"
reasoning_model: "gpt-4o"
agent_actions:
  - "Identify the full context of hash usage"
  - "Search for other weak crypto patterns in the codebase"
  - "Determine appropriate replacement based on use case"
safe_autofix: false
autofix_guidance: "Cannot autofix - requires understanding use case to select correct replacement algorithm"
review_questions:
  - "Is this hash used for security purposes (passwords, authentication)?"
  - "Is this for integrity verification (file checksums)?"
  - "What is the regulatory/compliance context?"
RAG_chunks:
  - "MD5 is broken for security purposes since 2004 (collision attacks)"
  - "SHA1 is broken for security since 2017 (SHAttered collision)"
  - "Replace with SHA-256/SHA-3 for integrity, bcrypt/Argon2 for passwords"
```

**Performance Budget**: 3ms/file (AST query)
**Complexity**: low

### CGR_SEC_CRY_003: weak-encryption

| Field | Value |
|-------|-------|
| **rule_id** | `CGR_SEC_CRY_003` |
| **concept_id** | `CC_SEC_CRY_003` |
| **domain** | Security |
| **category** | Cryptography |
| **languages** | rust, python, javascript, typescript, go, java, c, cpp |
| **severity** | CRITICAL |
| **precision target** | high |
| **sonarqube_equivalent** | S2254 |
| **cwe_ids** | CWE-327 |

**Detection Strategy**: Tree-sitter AST query
- Match import/use of DES, 3DES, RC2, RC4, ECB mode
- Pattern match algorithm names in crypto imports and instantiation

**required_keywords**:
```rust
layer! { "preflight" }
required_keywords! { "des", "rc4", "rc2", "3des", "ecb", "DES", "RC4", "ECB" }
```

**AST Query Patterns**:

*Rust (using `aes` crate)*:
```tree-sitter
(use_declaration
  path: (scoped_identifier
    name: (identifier) @name
    (#match? @name "^(des|3des|rc4|rc2)$")))
(call_expression
  function: (identifier) @func
  (#match? @func "^(new_des|new_rc4|new_rc2|des_encrypt|rc4_encrypt)$"))
```

*Python*:
```tree-sitter
(import_statement
  (dotted_name) @name
  (#match? @name "Crypto|Cipher"))
(assignment
  value: (call
    function: (attribute
      object: (identifier) @obj
      attr: (identifier) @attr)
    (#match? @obj "Crypto|Cipher")
    (#match? @attr "des|3des|rc4|ecb")))
```

*JavaScript*:
```tree-sitter
(call_expression
  function: (member_expression
    object: (identifier) @obj
    property: (identifier) @prop)
  (#match? @prop "createCipher|createDecipher|rc4"))
(call_expression
  function: (member_expression
    object: (identifier) @obj
    property: (identifier) @prop)
  (#match? @prop "createCipheriv|createDecipheriv")
  arguments: (arguments
    (string) @mode
    (#match? @mode "^(des|3des|rc[24]|ecb)$")))
```

**Preflight Checks**:
1. Check for ECB mode in block cipher configuration
2. Verify algorithm name in cipher instantiation
3. Skip cryptographic constants used in non-security contexts

**False-Positive Conditions**:
- Legacy protocol support (when required for interoperability)
- Test code with mock crypto
- Documentation examples

**Exclusions**:
- `#[cfg(test)]` contexts
- Comments/documents
- Non-security use cases explicitly documented

**Axiom Metadata**:
```yaml
event_kind: security.cryptography.weak_encryption
severity_mapping:
  CRITICAL: [high, critical]
fix_playbook: |
  1. Identify the encryption use case
  2. For DES/3DES: Replace with AES-256
  3. For RC4: Replace with AES-GCM or ChaCha20-Poly1305
  4. For ECB mode: Replace with GCM or CBC with proper IV
  5. Ensure key management is also updated
```

**agent_semantics**:
```yaml
summary: "Weak encryption algorithm detected - DES/RC4/ECB provide inadequate protection"
reasoning_model: "gpt-4o"
agent_actions:
  - "Identify the full encryption context (key size, mode, IV handling)"
  - "Search for other weak crypto in the codebase"
  - "Check key length (DES uses 56-bit, effectively weak)"
safe_autofix: false
autofix_guidance: "Cannot autofix - requires understanding encryption context and compatibility requirements"
review_questions:
  - "Is this for new development or legacy system integration?"
  - "What is the minimum compatibility requirement?"
  - "Is the data at rest or in transit?"
RAG_chunks:
  - "DES has 56-bit key space, brute-forceable since 1998"
  - "RC4 has biases exploitable in practice since 2013"
  - "ECB mode reveals patterns in encrypted data"
  - "Replace: AES-256-GCM for confidentiality + authenticity"
```

**Performance Budget**: 3ms/file (AST query)
**Complexity**: low

### CGR_SEC_CRY_005: insecure-tls-config

| Field | Value |
|-------|-------|
| **rule_id** | `CGR_SEC_CRY_005` |
| **concept_id** | `CC_SEC_CRY_005` |
| **domain** | Security |
| **category** | Network |
| **languages** | rust, python, javascript, typescript, go, java |
| **severity** | CRITICAL |
| **precision target** | medium |
| **sonarqube_equivalent** | S4834 |
| **cwe_ids** | CWE-295, CWE-296 |

**Detection Strategy**: Regex pattern on source lines
- Match insecure TLS configuration patterns
- Verify flag settings that disable certificate verification

**required_keywords**:
```rust
layer! { "preflight" }
required_keywords! { "verify", "ssl", "tls", "cert", "https" }
```

**Regex Patterns**:
```rust
// Word boundary-safe patterns for TLS verification disable
let patterns = [
    r"(?:\b|_)verify\s*[:=]\s*false",
    r"(?:\b|_)verify\s*[:=]\s*0",
    r"(?:\b|_)NoVerify\b",
    r"(?:\b|_)ALLOW_SELF_SIGNED\s*[:=]",
    r"accept_invalid_certs?\s*\(\s*true\s*\)",
    r"danger_accept_invalid_certs\s*\(\s*true\s*\)",
    r"ssl_verify\s*[:=]\s*false",
    r"tls_verify\s*[:=]\s*false",
    r"insecure\s*[:=]\s*true",
    r"request\(\s*\{.*verify\s*:\s*false",
    r"set_openssl_verify\s*\(\s*0\s*\)",
];
```

**Preflight Checks**:
1. Skip commented lines
2. Skip test/mock configurations
3. Check for test context indicators
4. Verify pattern is in configuration context (not just a comment)

**False-Positive Conditions**:
- Test/mock configurations
- Local development with explicit `#[cfg(dev)]` or `NODE_ENV=test`
- Documentation examples

**Exclusions**:
- `#[cfg(test)]`, `#[cfg(dev)]`, `NODE_ENV=test`
- Comment-only mentions
- Self-signed cert in test fixtures with clear indication

**Axiom Metadata**:
```yaml
event_kind: security.network.tls_insecure_config
severity_mapping:
  CRITICAL: [high, critical]
fix_playbook: |
  1. Identify the TLS configuration location
  2. Enable certificate verification (verify: true)
  3. Configure proper certificate chain validation
  4. For self-signed certs in dev: use devcert or mkcert
  5. For production: ensure proper CA bundle is configured
```

**agent_semantics**:
```yaml
summary: "Insecure TLS configuration detected - certificate verification is disabled"
reasoning_model: "gpt-4o"
agent_actions:
  - "Identify the full TLS configuration context"
  - "Check if this is in test, dev, or production code"
  - "Search for other insecure TLS patterns"
safe_autofix: false
autofix_guidance: "Cannot autofix - requires understanding of certificate infrastructure"
review_questions:
  - "Is this in test/development or production code?"
  - "Is there a legitimate reason (self-signed cert in dev)?"
  - "What is the proper certificate setup for this environment?"
RAG_chunks:
  - "Disabled TLS verification allows man-in-the-middle attacks"
  - "CWE-295: Improper Certificate Validation"
  - "CWE-296: Improper Following of a Certificate Chain"
  - "Fix: Configure proper CA certificates or use system trust store"
```

**Performance Budget**: 1ms/file (regex line scan)
**Complexity**: low

### CGR_SEC_CRY_006: deprecated-crypto-functions

| Field | Value |
|-------|-------|
| **rule_id** | `CGR_SEC_CRY_006` |
| **concept_id** | `CC_SEC_CRY_006` |
| **domain** | Security |
| **category** | Cryptography |
| **languages** | rust, python, javascript, typescript, go, java, c, cpp |
| **severity** | MAJOR |
| **precision target** | high |
| **sonarqube_equivalent** | S5122 |
| **cwe_ids** | CWE-327 |

**Detection Strategy**: Tree-sitter AST query
- Match deprecated crypto function patterns specific to each language

**required_keywords**:
```rust
layer! { "preflight" }
required_keywords! { "crypto", "cipher", "decipher", "hashlib", "subtle" }
```

**AST Query Patterns**:

*JavaScript/TypeScript (Node.js crypto)*:
```tree-sitter
(call_expression
  function: (member_expression
    object: (identifier) @obj
    property: (identifier) @prop)
  (#eq? @obj "crypto")
  (#any-of? @prop "createCipher" "createDecipher" "createCipheriv" "createDecipheriv"))
```

*Python (hashlib)*:
```tree-sitter
(assignment
  value: (call
    function: (attribute
      object: (identifier) @obj
      attr: (identifier) @attr)
    (#eq? @obj "hashlib")
    (#match? @attr "^(md5|sha1)$")))
```

*Java*:
```tree-sitter
(import_statement
  (scoped_identifier
    scope: (identifier) @scope
    name: (identifier) @name)
  (#eq? @scope "javax.crypto")
  (#eq? @name "Cipher"))
(call_expression
  function: (method_invocation
    object: (identifier) @obj
    name: (identifier) @method)
  (#eq? @method "getInstance")
  (arguments (string) @algo
    (#match? @algo "^DES$"))))
```

*C (OpenSSL legacy)*:
```tree-sitter
(call_expression
  function: (identifier) @func
  (#match? @func "^EVP_"))
```

**Preflight Checks**:
1. Check for deprecated API usage patterns
2. Skip test implementations
3. Check for security context

**False-Positive Conditions**:
- Legacy system interop with documented rationale
- Test code using deprecated APIs to test legacy integration
- Comments explaining deprecation and migration plan

**Exclusions**:
- `#[cfg(test)]` contexts
- Explicit migration comments
- Non-security use cases

**Axiom Metadata**:
```yaml
event_kind: security.cryptography.deprecated_api
severity_mapping:
  MAJOR: [medium, high]
fix_playbook: |
  1. Identify the deprecated function and its use case
  2. Consult language-specific crypto best practices
  3. For Node.js: migrate to crypto.subtle or Node.js crypto module updates
  4. For Python: use hashlib with modern algorithms
  5. For Java: use KeyGenerator with AES/GCM or GCMParameterSpec
  6. Run tests to verify functionality
```

**agent_semantics**:
```yaml
summary: "Deprecated cryptographic function detected - these APIs have known weaknesses or are being phased out"
reasoning_model: "gpt-4o"
agent_actions:
  - "Identify the deprecated function and its context"
  - "Check for migration guides in the codebase"
  - "Search for other deprecated crypto usage"
safe_autofix: false
autofix_guidance: "Cannot autofix - replacement depends on use case and language version"
review_questions:
  - "What is the replacement API for this language/framework?"
  - "Is there a migration guide available?"
  - "What is the timeline for deprecation?"
RAG_chunks:
  - "Deprecated crypto functions may have known vulnerabilities"
  - "Node.js crypto.createCipher deprecated since v0.8"
  - "Python hashlib.md5/sha1 for non-security use is acceptable"
  - "Java DESKeySpec deprecated - use AES"
  - "Fix: Use modern equivalent per language/framework best practices"
```

**Performance Budget**: 3ms/file (AST query)
**Complexity**: low
